<p><strong>Key Highlights</strong></p>
<ul>
<li><strong>Clear hardware evidence per device</strong> so you can see exactly how hardware details are discovered</li>
<li><strong>Expanded software evidence visibility</strong> beyond installers, including file‑based and OS‑level evidence</li>
<li><strong>Richer OS details</strong> with added context like OS caption and CSD version</li>
<li><strong>Consistent UI and API evidence</strong> for more reliable analysis and reporting</li>
<li><strong>Improved confidence in inventory data</strong> through greater transparency and traceability</li>
</ul>
<p><strong>Why It Matters</strong></p>
<p>Accurate IT decisions depend on understanding where your data comes from. These enhancements give you deeper visibility into the evidence behind device and software records, making it easier to validate inventory, troubleshoot discrepancies, and confidently act on IT Visibility insights across your environment.</p>

<h3>Explore massive inventory datasets instantly, without exports or limits</h3>
<p>Flexera One <a href="https://www.flexera.com/blog/it-visibility/empowering-users-with-intuitive-actionable-data-introducing-data-explorer/">Data Explorer</a> in IT Visibility now supports asynchronous rendering, allowing you to sort, filter, and search millions of rows of inventory data directly in the UI—without truncation, performance issues, or exporting data to external tools.</p>
<p><strong>Key Highlights</strong></p>
<ul>
<li>Work with millions of records in the UI: no more row or payload limits</li>
<li>Search, filter, and sort across the full dataset, not just visible rows</li>
<li>No forced exports: analyze large reports directly in Flexera One</li>
<li>Automatic performance optimization that kicks in seamlessly for large datasets</li>
<li>Consistent user experience with no changes to existing workflows</li>
</ul>
<p><strong>Why It Matters</strong></p>
<p>Large‑scale environments demand large‑scale analysis. This enhancement eases daily operational reporting by keeping even massive datasets fully usable inside Flexera One. The result is faster insights, less context switching, and greater trust in Flexera One as a single destination for enterprise‑grade inventory reporting and analytics.</p>

<h2>Innovation area: IT Asset Management</h2>
<h3>Bulk Deletion</h3>
<p>Data management in ITAM has been enhanced with the introduction of bulk deletion functionality for purchases and assets.</p>
<p><strong>Key highlights include</strong>:</p>
<ul>
<li>Users can now delete purchases and assets in batches of up to 1,000 records</li>
<li>Bulk deletion can also be performed utilizing the business adapter</li>
<li>Deletions are executed automatically during reconciliation after “Marked for Deletion” and do not degrade system performance</li>
</ul>
<p><strong>Why It Matters</strong></p>
<p>Previously, maintaining clean data was challenging due to the manual effort of deleting single entries. Data management and cleanup is essential for maintaining a coherent asset inventory, giving better visibility into assets that you’re able to take real cost saving actions on.</p>

<h3>VM density columns by operating system added in Report Builder</h3>
<p>Report builder has been enhanced to report on virtual machine (VM) density by operating system variants at the cluster level.</p>
<p><strong>Key highlights include</strong>:</p>
<ul>
<li>VM density is calculated as the ratio of VM’s to the number of hosts in a cluster</li>
<li>New columns have been added to support licensing scenarios
<ul>
<li>Windows virtual machines (density)</li>
<li>RHEL virtual machines (density)</li>
<li>SUSE virtual machines (density)</li>
</ul>
</li>
</ul>
<p><strong>Why It Matters</strong></p>
<p>These updates to report builder provide clearer insight into the distribution of VM’s by operating system relative to the number of hosts in a cluster. This helps our customers better understand VM density and the impact on licensing.</p>
<h2>Innovation area: SaaS Management</h2>
<h3>Optimize Microsoft 365 with Real Mailbox and OneDrive Usage Insight</h3>
<p>The Flexera M365 connector now includes Mailbox and OneDrive usage data, giving teams deeper insight into how Microsoft 365 storage is actually being consumed. This added visibility helps customers better manage costs, reduce sprawl, and strengthen governance decisions.</p>
<p><strong>Key Highlights</strong></p>
<ul>
<li><strong>See storage usage beyond licenses</strong><br />
Understand more than just who has a license with mailbox size, activity, and OneDrive consumption.</li>
<li><strong>Identify waste and risk faster</strong><br />
Spot inactive mailboxes, over‑utilized accounts, and growing OneDrive sprawl.</li>
<li><strong>Make smarter optimization decisions</strong><br />
Use real usage data to support cleanup, right‑sizing, and renewal conversations.</li>
<li><strong>Reduce manual reporting effort</strong><br />
Eliminate reliance on external tools and manual analysis for usage insights.</li>
</ul>
<p><strong>Why It Matters</strong></p>
<p>Microsoft 365 costs and risks are driven by licensing, but storage usage also plays a major role. By adding Mailbox and OneDrive usage visibility, Flexera helps organizations connect usage, cost, and governance data in one place, enabling more confident optimization decisions and stronger control over SaaS growth.</p>
<h2>Innovation area: Security Vulnerability Management</h2>
<h3>Stronger security, expanded macOS support, and greater deployment flexibility in the latest SVM Cloud update</h3>
<p>The SVM March 2026 cloud update delivers important fixes and enhancements that improve patch reliability, strengthen security posture, and expand deployment flexibility, while adding native support for macOS environments and more secure patch signing options.</p>
<p><strong>Key Highlights</strong></p>
<ul>
<li><strong>Reliable Adobe patching</strong> with fixed detection logic for Adobe Acrobat DC and Reader DC</li>
<li><strong>Stronger security and compliance</strong> by resolving 75 high and 50+ medium/low severity code scanning alerts</li>
<li><strong>Cleaner vulnerability posture</strong> with zero Dependabot vulnerabilities reported</li>
<li><strong>Expanded macOS support</strong> with native Jamf Pro integration for deploying .dmg and .pkg packages</li>
<li><strong>Greater deployment flexibility</strong> with support for multiple setup package URLs in the PFM API</li>
<li><strong>Flexible digital patch signing</strong> options, including standard and custom (HSM‑based) signing</li>
<li><strong>Seamless compatibility</strong> with the latest Patch Publisher (v7.31.2212)</li>
</ul>
<p><strong>Why It Matters</strong></p>
<p>Modern environments demand secure, reliable patching across diverse platforms. Deploy patches with greater confidence, reduce security risk, and simplify complex Windows and macOS deployments, without changing existing workflows. By combining stronger security, broader endpoint support, and more flexible automation, SVM continues to scale with your environment as complexity grows.</p>
